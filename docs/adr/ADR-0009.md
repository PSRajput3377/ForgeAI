# ADR-0009 — Agent system lives in packages/, imported by the API

- **Status:** Accepted
- **Phase:** 2

## Context
The AI engine could live inside `apps/api` or as standalone packages. We want it
reusable and independently testable, not owned by the web API.

## Decision
The engine (`core`, `models`, `prompts`, `tools`, `agents`) lives in repo-root
`packages/`. The API depends on it via `PYTHONPATH` (Dockerfile) and pytest
`pythonpath`.

## Consequences
- ✅ The API is one consumer of the engine, not its owner.
- ✅ Engine is unit-testable without the web layer.
- ➖ Requires path wiring; documented in `apps/api/README.md` and the Dockerfile.
