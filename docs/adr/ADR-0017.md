# ADR-0017 — Event-driven observability with offline backends

- **Status:** Accepted
- **Phase:** 6

## Context
Users need to see what ForgeAI is doing — live agent status, a replayable
timeline, metrics, and traces. We also need this testable without Langfuse or a
database, and without coupling agents to any UI or storage.

## Decision
Introduce an in-process **EventBus**: agents/tools/engine publish typed `Event`s;
subscribers (EventStore, MetricsCollector, Tracer, WebSocket connections)
consume them. The bus assigns a monotonic `tick` for ordering/replay.

- Instrumentation is **opt-in**: `build_workflow(..., bus=…)` wraps each node;
  with no bus the workflow is unchanged.
- Tracing is behind a `Tracer` interface: `NullTracer` (default) or
  `LangfuseTracer` (lazy import).
- Storage is an in-memory `EventStore` now; PostgreSQL (Events/AgentRuns/
  ToolCalls tables) drops in behind the same shape later.

## Consequences
- ✅ Producers and consumers are fully decoupled (agents don't know about UIs).
- ✅ Deterministic, replayable timelines; metrics derive from events alone.
- ✅ Entire system testable offline (extends ADR-0011/0015/0016).
- ✅ Live updates via WebSocket, no polling.
- ➖ In-process bus only (single API process); a cross-process broker (Redis
  pub/sub) would be needed to scale horizontally — deferred.
