# ADR-0004 — Three purpose-specific data stores

- **Status:** Accepted
- **Phase:** 0

## Context
Different data has different access patterns: durable relational records, fast
ephemeral coordination state, and semantic similarity search.

## Decision
- **PostgreSQL** — system of record (users, projects, sessions, tasks, messages).
- **Redis** — running jobs, agent queue, locks, temporary state.
- **Qdrant** — vector embeddings for memory and RAG.

## Consequences
- ✅ Each store does what it is best at.
- ➖ Three services to run and operate (mitigated by Docker Compose).
- **Rejected:** overloading Postgres for queues and vectors.
