# ADR-0003 — All LLM access goes through a Model Router

- **Status:** Accepted
- **Phase:** 0

## Context
Hardcoding a provider SDK into agents couples the whole system to one vendor
and makes per-role model choice impossible.

## Decision
Agents call a `ModelRouter.complete_for(role, messages)`. The router maps each
`AgentRole` to a configured model and forwards to an `LLMProvider`
implementation. Provider/model are configurable at runtime.

## Consequences
- ✅ Provider independence — add OpenAI/Claude/Gemini by writing one class.
- ✅ Per-role models (planner vs coder vs embed) configured in one place.
- ✅ Enables the offline `EchoProvider` (ADR-0011).
- ➖ One indirection layer between agent and LLM.
- **Rejected:** direct provider SDK calls inside agents.
