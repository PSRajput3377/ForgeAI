# ADR-0012 — Diagrams as Mermaid, not PNG

- **Status:** Accepted
- **Phase:** 2.5

## Context
The Phase 2.5 brief listed diagrams as `.png` files. Binary images in a code
repo can't be diffed or reviewed, drift from the code, and need a separate tool
to regenerate.

## Decision
Author diagrams as Mermaid (`.mmd`) source under `docs/diagrams/` and embed them
as ```` ```mermaid ```` blocks in the docs. PNG export remains possible via
`mermaid-cli` when a raster is genuinely needed.

## Consequences
- ✅ Diagrams are version-controlled, diffable, and render natively on GitHub.
- ✅ Single source of truth; no stale binaries.
- ➖ Complex hand-drawn illustrations aren't expressible — acceptable; ours are
  structural.
