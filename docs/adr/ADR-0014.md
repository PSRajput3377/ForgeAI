# ADR-0014 — Capability System: agents request capabilities, not tools

- **Status:** Accepted
- **Phase:** 3

## Context
If agents pick tools by name, agent logic becomes coupled to the tool catalog.
Replacing a tool (local search → web search) or having several tools satisfy the
same need would require editing every agent.

## Decision
Agents may declare a **capability** (`MODIFY_FILE`, `FIND_DOCUMENTATION`,
`RUN_SANDBOXED`, …). A `CapabilityResolver` maps each capability to an ordered
list of tool names and returns the first one currently registered. Agents may
still address a tool by name when they truly need a specific one.

## Consequences
- ✅ Implementations can change without touching agents.
- ✅ Multiple tools can satisfy one capability (with fallback order).
- ✅ New tools plug in by extending the capability map.
- ➖ One more indirection; the capability→tool map must be maintained.
- Builds on the registry (ADR-0006 of tools) and Tool Manager lifecycle.
