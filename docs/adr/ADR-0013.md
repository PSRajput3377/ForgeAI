# ADR-0013 — Separate specs/ (what) from docs/ (how)

- **Status:** Accepted
- **Phase:** 2.5

## Context
Documentation that mixes "how the system currently works" with "what a
component must do" makes it hard to verify an implementation against intent.

## Decision
- `docs/` explains **how** ForgeAI works today (architecture, agents, flows).
- `specs/` defines **what** each component must do (contracts, invariants,
  acceptance criteria) — the checkable target for implementations.

## Consequences
- ✅ Every implementation can be checked against a written spec before merge.
- ✅ Mirrors larger engineering orgs; makes the project feel production-grade.
- ➖ Two places to keep current — linked bidirectionally to reduce drift.
